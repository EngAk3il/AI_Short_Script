## THE CORE TRUTH

People don't watch Shorts because of information.

## THE 5-PHASE STRUCTURE (Works for ALL creators)

```
PHASE 1: STOP (0–5s)     → Why should I not scroll?
PHASE 2: TRAP (5–20s)    → Give me enough to need the answer
PHASE 3: BUILD (20–45s)  → Stack evidence. Make me feel smart.
PHASE 4: TWIST (45–60s)  → Reveal the real story / villain / insight
PHASE 5: CLOSE (60–80s)  → Resolution + emotional payoff
```

---

## WHAT EACH PHASE MUST DO

### PHASE 1 — STOP (0–5 seconds)
The thumb is moving. You have ONE job: stop it.

**Weapons that stop thumbs:**
- Ego challenge: "Tum galat ho." / "Tum game samajhe hi nahi."
- Paradox: "Jeet ke bhi haar gaye." / "Nuksaan jo actually win tha."
- Betrayal: "Jab tum trophy celebrate kar rahe the, investors exit kar rahe the."
- Question with obvious-but-wrong answer: "Aakhir unhone ₹10 crore kyun lagaye?"
- Number shock: "₹16,000 crore. Sirf cricket team ke liye."

**What kills PHASE 1:**
- "Aaj hum baat karenge..."
- "Hey guys, welcome back..."
- Starting with background/context
- Any sentence over 10 words in the first 5 seconds

---

### PHASE 2 — TRAP (5–20 seconds)
You stopped them. Now trap them.

**How:** Give partial information. Don't complete the picture yet.
Make the viewer feel: "Wait I need to know how this ends."

**Weapons:**
- Name the concept but don't explain: "Yeh Intellectual Property ka game hai."
- Show the scale: "45 million followers. 1 billion viewers."
- Name the villain without revealing them: "Bechne wala kaun tha? Uska naam sunouge toh shock ho jaouge."
- The "lekin" pivot: "Numbers yeh hain. Lekin yahan pe interesting baat yeh hai—"

---

### PHASE 3 — BUILD (20–45 seconds)
Stack evidence. Each line should make the viewer feel **smarter**.

**Rule:** Every fact must be a brick. Remove any fact that doesn't add to the wall.

**Pacing secret:** Alternate fast and slow.
- Fast = facts (rapid fire, staccato)
- Slow = meaning (let each insight land)

Example:
```
[FAST] 2015 mein ₹10 crore. 2025 mein ₹150 crore.
[SLOW] Sirf ek trophy se... yeh sab badal gaya.
[FAST] 45 million followers. 300% jersey spike. 1 billion viewers.
[SLOW] Ye sirf cricket nahi. Ye ek attention machine hai.
```

---

### PHASE 4 — TWIST (45–60 seconds)
This is the most important phase. This is WHY they watch till end.

The twist is NOT a plot twist. It's a **perspective shift**.

Examples:
- "Bechne waale ne peak par becha. Fans celebrate kar rahe the. Investors exit kar rahe the."
- "Yeh ₹8.5 crore ka nuksaan tha hi nahi. Yeh ek business pitch ki cost thi."
- "Bollywood waale gate rakhte the. Unhone ₹10 crore mein apna gate khareed liya."

**The twist should make viewer think:** "Oh. OH. I never thought of it that way."

**Director note:** SLOW DOWN here. Drop your voice. The viewer leans in.

---

### PHASE 5 — CLOSE (60–80 seconds)
Resolution. But not too neat.

The best closes leave ONE thought that the viewer carries with them.

**Per-creator close formulas:**
- **GenZway:** Hard verdict. No CTA. "Tum hamesha woh rahouge jinka attention bika ja raha hai."
- **Shivanshu:** "Ultimately, [philosophical truth]." Then subscribe CTA.
- **Finsaheli:** Personal lesson + "Mujhe follow karna mat bhulna."

# Script Generation Rules 📋

> These rules MUST be followed for EVERY script generated in this system.

## Rule 0: ZERO HALLUCINATION (ABSOLUTE)

> [!CAUTION]
> **NEVER fabricate, guess, or construct URLs, data points, statistics, or facts.**
> If you cannot find a verified source for a data point, say so explicitly.
> Getting something wrong wastes real human effort and breaks trust.

### What counts as hallucination:
- **Constructing URLs** by guessing the path pattern (e.g. `/article-slug-123.html`)
- **Inventing statistics** not backed by a verifiable source
- **Guessing institution names** (e.g. saying "IIT" when it's actually "IISc")
- **Making up article titles** or publication names
- **Assuming URL structures** for websites you haven't visited

### What you MUST do instead:
1. **Search the web** for the actual data point
2. **Click through** to the actual article in the browser
3. **Copy the exact URL** from the address bar after confirming the page loads
4. If a URL cannot be verified, **DO NOT include it** — leave a note: `[URL VERIFICATION NEEDED]`
5. If data cannot be sourced, **mark it clearly**: `[UNVERIFIED — needs manual check]`

### Priority 1: Link and Content Validation
- **Do not blindly trust a single provided link.** If a link is wrong, hallucinated, or contains contradictory info, the script will inherit those errors.
- **Cross-verify** claims with OTHER reputable sources before generating the script.
- If a provided link's content contradicts the *core premise* of the user's prompt (e.g., the link says "they won a trophy" but the prompt is strictly "without a trophy"), **prioritize the user's constraints** and find other valid sources that support the correct narrative, or omit the contradictory data completely.
