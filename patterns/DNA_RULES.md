# DNA Profile Writing Rules — V2 (Deep Immersion Standard)

> These rules define the exact standard for writing a Creator DNA Profile (`dna.md`).
> A DNA profile is the SINGLE SOURCE OF TRUTH before writing any script for a creator.
> Every claim in the DNA MUST be backed by exact quotes or counted frequencies from real transcripts.

---

## Pre-Requisite: Full Transcript Immersion

**Before writing a DNA profile, you MUST:**
1. Read EVERY single `transcript.txt` for the creator (no sampling, no skipping).
2. Count the total transcripts read. State this number at the top of the `dna.md`.
3. Take note of EXACT phrases, verbal tics, transition words, and sentence structures.
4. Identify ALL content formats the creator uses (e.g., solo explainer, interview clip, podcast teaser).

---

## DNA Profile Mandatory Sections

Every `dna.md` MUST contain ALL of the following 10 sections. No section may be skipped.

### Section 1: Creator Mindset (WHO they are)
- What is their CORE IDENTITY? Not "educational creator" — something deeper.
- What is their CORE BELIEF? Write it as a direct quote in their voice.
- What is their RELATIONSHIP with the audience? (Brother, teacher, friend, prosecutor, etc.)
- What are they NOT? (Define boundaries — e.g., "He is NOT a news anchor. He is NOT a business educator.")

### Section 2: Content Pillars (WHAT they talk about)
- List 4-6 content pillars as a TABLE with:
  - Pillar name
  - 2-3 REAL video examples (from actual transcripts)
  - Frequency (Very High / High / Medium / Low)
- NEVER use generic pillar names. Use specific ones backed by the data.

### Section 3: Voice Rules — Exact Speech Patterns (HOW they talk)
This is the MOST IMPORTANT section. Break into sub-categories:

#### ✅ SIGNATURE PATTERNS (used in 50%+ of videos)
- List every recurring verbal tic, connector, filler, transition phrase.
- For EACH pattern:
  - Write the EXACT phrase in Hindi/Hinglish/English as spoken.
  - Provide 2-3 REAL examples from transcripts showing it in context.
  - State approximate frequency (e.g., "appears in ~60% of videos").

#### ⚠️ SOMETIMES patterns (15-50% of videos)
- Same format as above but marked as "use sparingly".

#### ❌ NEVER DO
- List things the creator NEVER does, with evidence.
- Include tone violations (e.g., "NEVER aggressive"), vocabulary violations (e.g., "NEVER uses MBA jargon"), structural violations.

### Section 4: Hook Types — How They Open (EXACT opening lines)
- Classify hooks into 3-5 TYPES (not just "question" or "statement" — be specific).
- For EACH type:
  - Give it a descriptive name (e.g., "Specific Location + Shocking Fact")
  - Provide the approximate percentage frequency.
  - List 3-4 REAL opening lines from actual transcripts (verbatim).

> [!WARNING] **ANTI-REPETITION BINDING RULE**
> Each generated script MUST use a DIFFERENT hook type. Track previously used types and rotate.

### Section 5: Argument Structure — How They Build the Body
- Describe the EXACT step-by-step structure they follow in the body of their video.
- Use a numbered `Step 1 → Step 2 → ...` format.
- Include the TRANSITION PHRASES they use between steps.
- Provide any CRITICAL RULES (e.g., "Every claim must include a specific number").

### Section 6: Closing Types — How They End (EXACT closing lines)
- Classify closings into 3-5 TYPES.
- For EACH type:
  - Give it a descriptive name.
  - Provide the approximate percentage frequency.
  - List 2-3 REAL closing lines from actual transcripts (verbatim).

### Section 7: Pacing & Delivery
- Average WPS (Words Per Second) with range.
- Describe the ENERGY level (calm, aggressive, excited, measured, etc.).
- Note any pacing variations (e.g., "Solo explainers are faster than interview clips").
- Describe sentence structure (clean vs messy, overlapping thoughts vs structured).

### Section 8: Language Mix Deep-Dive
- Hindi/English percentage (from synthesis data).
- BUT go deeper: explain the PATTERN of how they mix languages.
- What stays in English? (Technical terms? Connector words? Full phrases?)
- What stays in Hindi? (Emotional phrases? Core narration?)
- Does the mix change by content type?

### Section 9: Emotional Arc
- Describe how the TONE shifts from start → middle → end.
- Use a one-line formula like: `[Curiosity] → [Wonder] → [Technical Depth] → [Awe]`
- State what emotion they NEVER express.

### Section 10: Special Formats / Sub-DNAs
- If the creator has multiple content FORMATS (e.g., solo vs interview, teaser vs full short), document each format's unique DNA separately.
- This prevents generating a "podcast clip" style script when the creator's solo content sounds completely different.

## Quality Checklist (Before finalizing a DNA)

- [ ] Every verbal tic has 2+ real transcript quotes as evidence
- [ ] Hook types have 3+ verbatim opening lines each
- [ ] Closing types have 2+ verbatim closing lines each
- [ ] Content pillars reference real video topics, not generic categories
- [ ] "NEVER DO" section is specific, not generic
- [ ] Language mix analysis goes beyond percentages
- [ ] Emotional arc is described, not just "positive" or "educational"
- [ ] Anti-repetition binding rule is included
- [ ] Total transcript count is stated at the top
- [ ] Updated date is stated at the top

## Anti-Hallucination Rules

1. **NEVER invent verbal tics.** If you can't find it in the transcripts, don't write it.
2. **NEVER use generic phrases** like "engaging opening" or "informative tone". Be SPECIFIC.
3. **ALWAYS provide the REAL quote** from the transcript. If you can't, mark it as "needs verification".
4. **NEVER assume** a creator uses a phrase just because it's common in Hindi YouTube. Check the data.
5. **When in doubt, UNDER-CLAIM.** It's better to list fewer patterns that are 100% accurate than many that are guesses.

## Section 11: Dynamic Generation Rules (MANDATORY FOR EVERY DNA)

> [!CAUTION] **THIS IS THE MOST IMPORTANT SECTION FOR SCRIPT GENERATION**
> The DNA is NOT a template to copy-paste from. It is a MINDSET MAP.
> Every script must feel like a FRESH, ORIGINAL creation by the creator — not a remix of the examples listed above.

### 11.1 — The DNA is a BOUNDARY, Not a Template

The DNA profile defines:
- ✅ **What the creator WOULD say** (the boundary)
- ✅ **How the creator THINKS** (the mindset)
- ✅ **What the creator NEVER does** (the guardrails)
- ❌ **NOT the exact words of every script** (that's copying, not creating)

**RULE:** When generating a script, you should be CHANNELLING the creator's mind — thinking like them, not quoting them. The verbal tics and transition phrases should appear NATURALLY (not forced), and the topic + angle + structure should be UNIQUE every time.

### 11.2 — Mandatory Variability Dimensions

For EVERY script generation, the following 6 dimensions MUST be varied:

| Dimension | What to Rotate | Never Do |
|-----------|---------------|----------|
| **1. Topic** | Pick a NEW topic the creator hasn't covered but WOULD cover based on their content pillars | Don't reuse a topic from the DNA examples |
| **2. Hook Type** | Use a DIFFERENT hook type from the last script generated | Never use the same hook type twice in a row |
| **3. Argument Variation** | Follow the creator's argument STRUCTURE but vary the number of steps, examples, and transitions | Never use the exact same step count or example pattern |
| **4. Verbal Tic Distribution** | Use 2-4 of the creator's signature tics per script, but ROTATE which ones appear prominently | Never use ALL signature tics in a single script (that sounds robotic) |
| **5. Closing Type** | Use a DIFFERENT closing type from the last script | Never default to the same CTA every time |
| **6. Emotional Intensity** | Vary the emotional energy (some scripts should be calmer, some more intense, even within the same creator) | Never make every script feel the same intensity |

### 11.3 — Verbal Tic Usage Rules

The DNA lists verbal tics with frequencies. Use this to determine NATURAL distribution:

- **Signature tics (50%+ frequency):** Should appear in MOST scripts but not ALL. Use 2-3 per script. Vary placement (beginning, middle, end).
- **Sometimes tics (15-50%):** Should appear in ~1 out of every 3 scripts. When they appear, use only 1-2 instances.
- **Never-do patterns:** ABSOLUTELY NEVER use. These are hard guardrails.

**CRITICAL:** Verbal tics should feel UNCONSCIOUS, not inserted. They should appear where the creator would NATURALLY say them during their flow of thought — not placed at regular intervals like a template.

### 11.4 — Topic Generation Rules

When generating a script, the topic MUST be:
1. **NEW** — Not one of the examples listed in the Content Pillars table
2. **WITHIN BOUNDS** — Falls within one of the creator's content pillars
3. **TIMELY** — If the creator covers current events, the topic should feel current (not dated)
4. **SPECIFIC** — Not generic ("AI is changing the world") but specific ("Claude's new computer-use feature lets AI control your mouse")

**Topic selection process:**
1. Check which content pillar hasn't been used recently
2. Find a REAL topic/news/fact that fits that pillar
3. Apply the creator's UNIQUE ANGLE (e.g., GenZway would expose the scam angle, designbyarpit would find the design insight, Shivanshu would tell the origin story)

### 11.5 — Structure Variation Rules

The argument structure in Section 5 is a SKELETON, not a rigid template:

- **Vary the step count:** If the structure says 5 steps, sometimes use 4, sometimes 6.
- **Vary transitions:** Use DIFFERENT transition phrases each time, drawn from the creator's vocabulary but not always the same one.
- **Vary depth:** Some scripts should go deep on one point; others should touch 3-4 points lightly.
- **Vary pacing:** Some scripts start slow and accelerate; others hit hard from the first word.

### 11.6 — The "Would They Say This?" Test

Before finalizing any script, run this mental test on EVERY line:

1. **Does this sound like something the creator would say out loud?** (Not write, not prepare — SAY.)
2. **Would a regular viewer recognize this as [creator name]'s voice?** (Without seeing who posted it.)
3. **Is there any line that sounds like a DIFFERENT creator?** (e.g., a GenZway script should NEVER have a line that sounds like hellooipsita's therapy reframe.)
4. **Are the verbal tics placed where the creator would NATURALLY say them?** (Not forced in.)
5. **Is the energy level correct?** (GenZway = 8/10 aggressive. hellooipsita = 4/10 calm. designbyarpit = 5/10 measured.)

## Script Generation Prompt Template

When generating a script for any creator, use this mental framework:

> [!IMPORTANT] **STEP 0: THE CREATOR ANGLE FILTER (MANDATORY)**
> BEFORE writing a single line of script, you MUST state the Creator Angle.
> You are FORBIDDEN from writing generic "news summaries" or "Wikipedia recaps."
>
> Ask yourself: "How does this creator specifically view this topic?"
> - If Shivanshu: Find the hidden origin story or strategic pivot. (Not the box office numbers, but the real-life spy who inspired it).
> - If GenZway: Find the hypocrisy, the scam, or the PR manipulation.
> - If hellooipsita: Find the psychological trap or the deeper emotion behind the trend.
> - If SciCoLens: Find the specific scientific mechanism that makes it possible.
>
> Write down: **"CREATOR ANGLE: [Explain the specific hidden story/angle you will take]"** before generating the script table.

### Step 1: Framework Setup
```
I am [CREATOR NAME]. I've never done a video on [TOPIC] before, but it's
exactly the kind of thing I'd cover because [REASON based on content pillars].

CREATOR ANGLE: [The specific hidden story / psychology / mechanism I am focusing on. NOT a summary.]

My hook type for this one will be: [DIFFERENT hook type from last time].
My closing type for this one will be: [DIFFERENT closing type from last time].

I'll use my natural speech patterns — "X" and "Y" verbal tics will come out,
but I'm not going to force them. They'll appear where they naturally fit.
```

> [!IMPORTANT] **STEP 2: THE SELF-CORRECTION AUDIT (MANDATORY)**
> After generating the script internally, but BEFORE presenting it to the user, you MUST run a self-check and fix yourself.
> 
> Ask yourself:
> 1. **The 'Wikipedia' Test:** Does this script just summarize facts a user could google? (If yes, REWRITE. It must be a specific angle/story).
> 2. **The 'Robotic' Test:** Did I stack too many DNA verbal tics unnaturally? (If yes, REMOVE SOME).
> 3. **The 'Hallucination' Test:** Did I state something as an absolute fact that is actually just a theory or unconfirmed? (If yes, FIX IT using phrases like "log maante hain" or "it is compared to").
> 
> If the script fails any of these audits, **fix it yourself** before saving the final `.md` file.

> [!CAUTION] **GENERATION TRACKING**
> After generating EACH script, note which hook type, closing type, and prominent verbal tics were used.
> The NEXT script for this creator MUST use DIFFERENT combinations.
